## 1. 项目概述

### 1.1 产品定位
搭配功能沙发成套系统使用的手机APP，适配高端单人位、多人位功能沙发。通过蓝牙与沙发主控盒通讯，扩展与智能沙发的交互方式。

### 1.2 目标平台
- iOS、Android、鸿蒙
- 兼容当前主流手机蓝牙版本

### 1.3 开发节奏（敏捷迭代）
| 节点 | 交付内容 |
|------|----------|
| 5月22日 | 第一版APP |
| 5月29日 | 第二版APP |
| 6月5日 | 第三版APP（演示版） |
| 后续 | 持续迭代优化 |

### 1.4 6月演示版硬件配置
> 第一版针对以下固定配置开发演示（**单人位**）：
- **座位推杆** + **头枕推杆**
- **坐垫通风**（1路，a配置：速冷/慢冷）
- **坐垫加热**（1路，a配置：速热/慢热）
- **单点控制气囊按摩**（背部，b组配置）
- **音律系统**（振子+音效+灯光）

> 按摩b组支持模式：Wave（波浪）、Catwalk（猫步）、Butterfly（蝶式）、Acupressure（点按）、Pat（拍打）

## 2. 系统架构

```
手机APP  ←→  蓝牙BLE  ←→  主控盒  ←→  气囊控制盒/各功能模块
```

- 主控盒：基于DY024改造，增加蓝牙模块
- 通讯协议：自定义蓝牙串口协议（详见第7章）
- APP需根据设备协议返回的**配置信息动态显示功能**（例如：若某沙发无腰托推杆，则不显示腰托控制）

## 3. 功能模块

### 3.1 登录（第一版已删除）
- ~~用户注册/登录~~ → 第一版**删除登录页**，直接进入设备连接/主界面
- 采用游客模式，无需服务器

### 3.2 设备连接

#### 3.2.1 首次使用流程
1. **NFC触碰**（支持NFC的机型）：
   - 第一次触碰 → 触发App下载/打开App，并自动连接蓝牙
   - 第二次及之后 → 快捷打开App并自动连接蓝牙
2. **无NFC机型**：
   - 进入沙发连接页面，自动扫描沙发蓝牙设备
   - 列表显示：可连接设备（沙发） / 其它设备
   - 点击设备即可连接
3. **扫码连接**（Demo阶段）：
   - 一机一码，扫描产品二维码自动连接对应蓝牙
   - 仅做功能演示，后续根据客户生产流程调整

#### 3.2.2 设备列表页
- 已连接过的设备直接显示在列表
- 显示产品图标、型号名称
- 点击产品图标快速连接
- 右上角 `+` 进入添加设备

#### 3.2.3 添加设备页
- 选择沙发型号（展示6个型号图标）：Recliner Plus / Recliner Pro / Power Recliner / loveseat Recliner / Smart Recliner 等
- 或：扫码添加 / 手动蓝牙搜索添加
- 搜索中动画 + 附近设备列表

#### 3.3.1 Posture（姿态控制）

**位置快捷键**（使用频率高，放在推杆控制上方）：
| 图标 | 名称 | 说明 |
|------|------|------|
| 🏠 | Home | 一键收回（全部推杆归位） |
| 📺 | TV | TV姿 |
| 🌙 | ZG | 零重力姿 |
| 💾 | Memory | 记忆位（短按执行，长按保存当前位置） |

> 注：若沙发铁架不支持零重力，则"ZG"替换为"躺姿(Recline)"

**推杆控制**（根据设备配置动态显示）：
- 座位推杆 (Seat) — 上升 / 下降
- 头枕推杆 (Head) — 上升 / 下降
- 腰托推杆 (Lumbar) — 上升 / 下降
- 助起推杆 (Foot) — 上升 / 下降
- 靠背推杆 (Back) — 上升 / 下降

> 控制方式：点击按钮持续控制，松开停止（有刷推杆无位置反馈，不做百分比条）

**多座椅模式**：
- 支持双人位/三人位沙发
- 顶部显示当前操控的座位名称（如"左边椅"、"右边椅"）

- 6月演示版先做单人位

## 4. UI/UX 规范

### 4.1 设计语言
- 简洁、现代、高端感
- 主色调：蓝色系（#0066CC 类似）
- 圆角卡片、阴影层次
- 白色/浅灰背景

### 4.2 图标规范
- 使用线性/面性结合的风格
- 各功能模块有独立图标
- 选中状态：蓝色填充背景 + 白色图标
- 未选中状态：灰色图标

### 4.3 交互动效
- 页面切换：滑动过渡
- 按钮点击：缩放反馈
- 推杆控制：按钮按压持续触发，松开停止
- 记忆位：长按0.8s触发保存，短按触发执行
- 沙发模型：前期用多张图片模拟动画，3D动画后续迭代

### 4.4 适配要求
- 移动端优先（竖屏）
- 适配不同屏幕尺寸
- 多推杆配置下，小屏手机需确保位置快捷键可见

## 6. 蓝牙通讯协议

### 6.1 帧格式

**APP → 设备（请求帧）**
| 字段 | 长度 | 内容 |
|------|------|------|
| 帧头 | 5B | 0xAA 0x21 0x01 0x31 0x01 |
| 命令码 | 2B | 0xXX 0xXX |
| 数据长度 | 2B | 0x00 0x04（数据包长度） |
| 数据包 | 5B | 见下发指令 |
| 校验和 | 1B | 帧头至数据包的算术累加和 |

**设备 → APP（应答帧）**
| 字段 | 长度 | 内容 |
|------|------|------|
| 帧头 | 5B | 0xBB 0x31 0x01 0x21 0x01 |
| 命令码 | 2B | 0xXX 0xXX |
| 数据长度 | 2B | 0x00 0xXX |
| 数据包 | 2+n B | 状态码(2B) + 返回信息 |
| 校验和 | 1B | 帧头至数据包的算术累加和 |

### 6.3 返回信息（状态上报）

设备周期性或在状态变化时向APP上报完整状态，数据包格式：

```
电机数量(1B) + [电机1速度(2B) + 电机1位置(2B) + 电机1位置类型(1B) + 电机1总行程(2B)] * n +
按摩数量(1B) + [按摩1模式(1B) + 按摩1力度(1B) + 按摩1剩余时间(2B)] * n +
加热数量(1B) + [当前加热模式(1B) + 加热1档位(1B) + 加热1剩余时间(2B)] * n +
通风数量(1B) + [当前通风模式(1B) + 通风1档位(1B) + 通风1剩余时间(2B)] * n +
灯带数量(1B) + [灯带1模式(1B) + 灯带1颜色(3B)] * n +
追腰数量(1B) + [追腰1模式(1B) + 追腰1力度(1B)] * n +
振子数量(1B) + [振子1模式(1B) + 振子1强度(1B)] * n +
音效数量(1B) + [音效1模式(1B) + 高音1增益(1B) + 低音1增益(1B) + 音量1(1B)] * n +
蓝牙配对(2B) + 2.4G配对(2B) + 蓝牙组网(2B) +
语言数量(1B) + [语言1(1B)] * n
```

> 数量字段为0时，该模块后续数据不存在。

## 7. 数据模型

### 7.1 设备模型
```typescript
interface Device {
  id: string;              // 设备唯一ID（蓝牙MAC或SN）
  name: string;            // 设备名称（如"Smart Recliner Pro"）
  model: string;           // 型号（如"CS-204-X"）
  image: string;           // 产品图
  connected: boolean;      // 连接状态
  pairedAt: number;        // 配对时间戳
  config: DeviceConfig;    // 设备配置（决定显示哪些功能）
}

interface DeviceConfig {
  motors: MotorConfig[];       // 推杆配置列表
  massageModes: number[];      // 支持的按摩模式列表
  heatingZones: number[];      // 加热区域数量及位置
  ventilationZones: number[];  // 通风区域数量及位置
  hasLight: boolean;           // 是否有灯光
  hasVibro: boolean;           // 是否有振子
  hasAudio: boolean;           // 是否有音律系统
  hasWaist: boolean;           // 是否有追腰
}

interface MotorConfig {
  type: 'seat' | 'head' | 'lumbar' | 'foot' | 'back';
  name: string;
  hasPositionFeedback: boolean;  // 是否有位置反馈
}
```

## 8. 非功能需求

### 8.1 性能
- APP启动时间 < 3s
- 蓝牙连接时间 < 5s
- 指令下发到设备响应 < 500ms
- 状态刷新频率：推杆控制期间高频（200ms），常态低频（2s）

### 8.2 稳定性
- 蓝牙断开后自动重连（最多3次）
- 后台保持连接（系统允许范围内）
- 操作指令失败时给出明确提示

### 8.3 安全
- 童锁功能：开启后禁用所有控制
- 推杆速度限制：无刷最低40%
